# Naming Conventions

## Files and Directories

- **kebab-case** for file names: `guest-profile.tsx`, `analytics-dashboard.ts`
- **PascalCase** for React components: `GuestProfile.tsx`, `AnalyticsDashboard.tsx`
- **snake_case** for Python files: `guest_profile.py`, `analytics_dashboard.py`

## Branches

```
feat/#<issue>-<short-description>
fix/#<issue>-<short-description>
spike/#<issue>-<short-description>
chore/#<issue>-<short-description>
```

## GitHub Issues

- Title: `[area] Short description`
- Areas: `prototype`, `docs`, `infra`, `analytics`, `commerce`, `ux`, `compliance`

## Variables and Functions

- **camelCase** for JavaScript/TypeScript
- **snake_case** for Python
- Descriptive names — no single-letter variables except loop counters
